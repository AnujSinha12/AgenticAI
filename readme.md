# Country Economic Indicator Agent

## Overview

**Country Economic Indicator Agent** is an Agentic AI application built using Generative AI technologies to provide economic insights across countries. The application leverages **LangChain**, **LangGraph**, **Azure OpenAI**, **Streamlit**, and **Matplotlib** to deliver conversational analysis of economic indicators such as exchange rates, inflation rates, and funds rates.

The application follows an AI Copilot-style user experience, enabling users to ask economic questions in natural language while receiving grounded, data-driven responses.

---


# App

The **App** layer is responsible for the user interface and user interaction.

## Features

### Streamlit-Based UI

The application uses **Streamlit** to provide an interactive web interface.

Main application file:

```text
app/
└── streamlit_app.py
```

### Copilot-Style Experience

The interface follows a similar interaction pattern to Microsoft Copilot Studio and consists of:

#### Chat Panel

* Allows users to interact with the AI agent using natural language.
* Displays streaming responses generated by the agent.

#### Chat History Sidebar

* Stores and displays previous conversations.
* Enables users to revisit earlier interactions.

#### Tools Panel

* Displays the LangChain/LangGraph tools invoked during execution.
* Improves transparency by showing how the agent arrives at its answers.
* Helps users understand which data sources are used for response generation.

# Agent

The intelligence layer of the application is implemented using **Agentic AI principles** with **LangChain** and **LangGraph**.

## Agent Framework

The agent is built using:

* LangChain
* LangGraph

## Available Tools

The agent can invoke the following tools:

### fetch_exchange_rate

Retrieves the latest exchange rate between a base currency and a target currency.

### fetch_inflation_rate

Retrieves inflation data from locally stored datasets.

### fetch_funds_rate

Retrieves funds rate data from locally stored datasets.

---

## LangGraph Workflow

The agent execution is modeled as a graph.

### Nodes

* LLM Node
* should_continue Node

### Conditional Routing

The graph uses conditional edges to determine:

* Whether a tool should be called
* Whether execution should continue
* When a final response should be generated

Example flow:

```text
START
  │
  ▼
LLM Node
  │
  ▼
should_continue
  │
  ├── Tool Required ──► Tool Node ──► LLM Node
  │
  └── Final Answer ───► END
```

---

## Short-Term Memory

The agent maintains conversation context using short-term memory.

Benefits include:

* Context-aware conversations
* Follow-up question handling
* Reference to previous user interactions
* Improved conversational continuity

Example:

```text
User: What is the inflation rate of the United States?

User: How does it compare to Canada?
```

The second question can be answered without repeating the original context.

---

## Streaming Responses

The agent supports streaming responses.

Benefits:

* Faster perceived response time
* Improved user experience
* Similar interaction pattern to Microsoft Copilot and ChatGPT

Instead of waiting for the complete response, tokens are streamed progressively to the user interface.

# Azure

The application uses Azure AI services for Large Language Model inference.

## Azure OpenAI

The LLM is deployed through **Azure AI Foundry**.

### Model Access

The application uses:

```python
AzureChatOpenAI
```

to connect to the deployed Azure OpenAI model.

---

## Secure Credential Management

Sensitive information is not stored directly in Python source files.

Configuration is managed through a `.env` file.

Example:

```env
AZURE_OPENAI_ENDPOINT=your_endpoint
AZURE_OPENAI_API_KEY=your_api_key
AZURE_OPENAI_DEPLOYMENT_NAME=your_deployment
AZURE_OPENAI_API_VERSION=your_api_version
```

Benefits:

* Improved security
* Easier environment management
* Better deployment practices

## Grounded Responses

The agent generates grounded responses using tool outputs whenever economic data is required.

Advantages:

* Reduced hallucinations
* Data-backed answers
* Improved trustworthiness
* Consistent economic analysis

Example:

```text
User: What is the inflation trend in Germany?

Agent:
1. Calls fetch_inflation_rate
2. Retrieves data from Excel
3. Generates an explanation based on actual values
```
